Ghost of Yotei — Ambient Lighting Pipeline Flowchart

Offline Bake Octree Runtime Interior Mask Octmap Relight RTGI Pipeline Pixel Output Memory Buffer Fallback
═══ PHASE 1: OFFLINE BAKE ═══

A 八叉树构建 (Octree Construction) Offline

输入: 美术 OBB 包围盒
美术定义定向包围盒覆盖需精细光照区域
切分为根节点 (示例 16m)
按美术指定大小均匀分割
递归细分至最小尺寸 (2m)
  • 仅包含几何的节点细分
  • 几何略微膨胀确保保守细分
  • 强制约束相邻节点尺寸差 ≤ 1 级
节点角点生成 Ambient Probe
每个节点的 8 个角点各放置一个 probe
嵌入几何检测 & 剔除
  • 双面渲染 probe 周围 G-Buffer
  • 大多数法线指向远离方向 → 判定嵌入 → 剔除
  • 记录相邻 probe 列表供运行时填充
输出: Octree 结构 + Probe 位置
自动每周重烘 2 次，小改无需美术介入
═══▶ probe 位置信息 ═══▶

B Probe Octahedral G-Buffer 烘焙 Offline

双面渲染 Probe 周围几何
  • 分辨率: 八叉树 16×16 / 世界网格 8×8
  • BC7 压缩存储
  • 不翻转背面法线
G-Buffer 通道布局
Albedo3 Bytes Translucency0.5 B Normal (Oct)2 B Surface Distance1 B Interior Mask0.5 B Sky Visibility1 B
Translucency 与 Interior Mask 互斥共享 1B:
内部 $[0.5, 1]$，外部 translucency $[0, 0.5]$
立体角 SH 权重预计算 (Python)
Texel → 立体角权重
每个 texel 近似为两个三角面:
$$\Omega = 2\arctan\!\left(\frac{|\mathbf{a}\cdot(\mathbf{b}\times\mathbf{c})|}{1+\mathbf{a}\cdot\mathbf{b}+\mathbf{a}\cdot\mathbf{c}+\mathbf{b}\cdot\mathbf{c}}\right)$$
输出: Baked Octahedral G-Buffers + 立体角 LUT
烘焙几何和材质而非光照 → 运行时 relighting 可响应动态光源变化
═══▶ interior 数据 ═══▶

C Interior Masking GPU Runtime

Interior Volume Geometry
输入: 美术制作的水密网格
建筑墙体内侧, 类 stencil shadow volume
Stencil Pass (双面渲染)
  • 绑定深度缓冲 + 全新模板缓冲
  • 不绑 RT，仅写 Stencil
  • 前面 depth fail → stencil +1
  • 背面 depth fail → stencil −1
判定: stencil=0 外部 / ≠0 内部
支持非凸形状
其他标记来源
多源叠加
  • 整对象标记 / Shader 参数
  • Deferred Decal / 顶点画刷 (门窗过渡)
动态对象 (角色)
CPU 射线查询 CPU
  • 获取角色所在八叉树节点
  • 每帧 1 根射线测 8 个 probe 可见性
  • Trilinear 混合 + 帧间混合抑制 popping
输出: Per-pixel Interior Mask
驱动: probe 选择、灯光过滤、风效、玩法
═══ PHASE 2: RUNTIME OCTREE SAMPLING & RELIGHTING ═══

D 运行时八叉树采样 (Octree Lookup) GPU

插值策略
同尺寸节点: Trilinear 混合
8 个角点 probe 直接三线性插值
跨尺寸节点: 渐进 Blend 到父节点
  • 位字段记录每面/边/角是否通向更大节点
  • 约束: 相邻尺寸差 ≤ 1 级
法线偏置权重
降低表面背后 probe 的权重，减少跨墙漏光
边缘 & 多八叉树
边缘渐隐 → 全局 probe 网格
八叉树边缘处回退到世界网格 (8×8 分辨率)
多八叉树重叠 → 取均值混合
拼出洞穴等不规则空间
输出: 插值后 Probe 索引 + 权重
供 Relighting 和 RTGI Miss fallback
═══▶ probe data ═══▶

E Octmap Runtime Relighting GPU~1200 probes/frame

更新调度
优先级队列
  • 采样时打标记 (sampled_this_frame)
  • 下帧更新: 超 1 帧未更新 + 被采样的 probe
  • 平均 ~1200/帧，最坏 12000 全量
  • 余量更新更老的 probe
  • 相机切换 / 新加载 → 强制全量 relight
Per-Texel 光照计算
每 Texel 视作真实表面
  • Distance 通道 → 重建世界空间位置
  • Albedo + Normal → 执行完整光照
  • 方向光: 直接采样阴影级联
  • 局部光源: 逐一贡献
Fallback: Texel 落在视锥外
→ 回退到附近 probe 天空可见性方向值；缓存结果避免 popping
SH 转换输出
Lit Texels → Spherical Harmonics
  • 使用预计算立体角权重累积
  • Interior/Exterior 分别累积两套 SH
输出: Per-Probe Irradiance SH (1.75 MB)
供像素评估三线性插值 + RTGI ray-miss 回退
═══ PHASE 3: RT RESOURCE PREPARATION ═══

F BVH 构建 (Acceleration Structure) CPUGPU

几何筛选
BLAS 池 (235 MB)
  • 仅纳入相机 20m 外的 LOD1+ 几何
  • ✅ 角色、动画布料、地形、大部分静态几何
  • ❌ 植被顶点动画、非角色 cutout、程序化草/实例、地形洞、水
剔除流程
  • 初始: CPU 基于 BVH 包围盒剔除
  • 最终: 迁移到 GPU (更多内存，更好性能)
TLAS 构建 (16 MB)
Top-Level Acceleration Structure → 指向所有可见 BLAS 实例
Instance Buffer (7.5 MB)
每实例包含transform + BLAS ref + swatch index
程序化几何替代
Imposter → 地形 Diffuse
程序化草/实例 → imposter 纹理烘到地形 diffuse 近似
═══▶ hit 时查询 ═══▶

G Material Swatch Atlas (25 MB) GPU

灵感: Bryan McPhail (Santa Monica Studio)
Swatch 渲染
  • 每 swatch = 32×32 mini G-Buffer (albedo/translucency/alpha)
  • Vertex Shader: UV 作为输出位置 (摊平到 UV 空间)
  • Pixel Shader: 正常 deferred 路径 (层混合/UV变换)
  • 保守光栅化确保亚像素三角形输出
  • 渲染后 dilate 防止 bilinear 越界
共享策略
  • 每 mesh + shader 组合共享一份 swatch
  • 方向相关 shader → 按实例独立烘焙
  • 也用于角色 cutout (头发/帽子)
Atlas 参数
上限5,120 swatches 尺寸32×32 per swatch 总内存~25 MB 渲染耗时数 μs / swatch 缓存命中大多数帧命中
输出: Swatch Atlas Texture
RT hit 时通过 instance buffer swatch index 查询漫反射材质
═══ PHASE 4: RTGI 7-STEP PIPELINE (Steps 1–4) ═══

1 Spawn Screen-Space Probes GPU

Probe 生成规则
  • 屏幕分 16×16 tile
  • 每 tile 随机选 8×8 象限一个像素 → 生成 probe
  • 其余象限比较 "覆盖度" (平面距离 + 法线差异)
  • 覆盖差 → 新增 probe
Screen Probes (3.5 MB)
上限32,000 1440p 平均2–3 / tile 每 probe64 rays, 8×8 oct
▶▶▶

2 Ray-Guided Sampling GPU

重要性采样
  • PDF = 上一帧 octahedral irradiance map
  • 无历史 → 回退八叉树 octmap
方向选取 (64 线程组)
  • 每线程: luminance × N·L
  • InclusivePrefixSum → CDF
  • 随机数 → 二分查找定位方向
输出: 64 ray directions / probe
偏向上一帧亮度高的区域
▶▶▶

3 Cast Rays GPU~2M rays

光线投射
  • 每 probe 64 根光线
  • 最大距离 4m
  • animated 表面增大 min hit distance
  • 最坏 ~2,000,000 rays/frame
超出 4m → Fallback
回退世界空间 probe SH (不做远距离 RT)
Shadow Ray 入队 (8 MB)
击中点超出阴影级联 → 入队 shadow ray
Ray Hits Buffer (32 MB)
存储hit pos + normal + instance ID Miss 标记特殊值表示未命中
▶▶▶

4 Light Rays GPU

Hit 路径: 着色命中点
  • Swatch Atlas → 漫反射材质
  • 八叉树 probe irradiance → 间接光
  • ≈ 2 次反弹 GI
  • 方向光 + 阴影贡献
Miss 路径
  • 回退八叉树 / 网格 irradiance
  • 采样位置 = probe 原点 (非射线终点)
  • 防止漏光
中间光照缓冲 (32 MB)
存储per-ray radiance (RGB) 用途供后续滤波累积
═══ PHASE 4 (cont.): RTGI PIPELINE (Steps 5–7) ═══

5 Spatial Filter GPU

空间滤波 (邻域去噪)
  • 每 probe 搜索 7×7 tile 邻域
  • 49 线程, 每线程一 tile 寻找相似 probe
  • 相似性: 位置 + 法线 + 深度
  • 用 hit distance 重投影命中点位置
  • Atomic 写回任意 texel → 保留细节
输出: 去噪后的 per-probe octahedral radiance
▶▶▶

6 History Filter (Probe 级) GPU

时间滤波 (帧间稳定)
AMD GI-1.0 [Boissé 2022] 思路:
$$w = \frac{|L_\text{cur} - L_\text{hist}|}{\max(L_\text{cur}, L_\text{hist})}$$
  • Lane intrinsics: probe 内所有 texel 取平均 history weight
  • 避免单 texel 噪声破坏整体历史
转换为 SH
滤波完成后累积 octahedral radiance → SH 系数
SH 结果 (1.75 MB)
格式L2 SH per probe 用途最终像素插值
▶▶▶

7 Final Per-Pixel Evaluation GPU

像素分配
  • 屏幕分 8×8 tile
  • 预算 min/max 深度
  • 预判八叉树影响 + 是否需 RTGI
距离混合
  • 80m 内: 使用 RTGI probe SH
  • 80m 外: 混入八叉树 octmap SH
像素级时间滤波
  • Luminance-delta 历史滤波
  • 剩余噪声 → FSR3 / PSSR
输出: Final Ambient Diffuse Irradiance
写入光照缓冲，参与最终合成
═══ PHASE 5: EDGE CASES & SPECIAL HANDLING ═══

H 多级回退 (Fallback Hierarchy) Fallback

Level 1: 像素无强相关 probe
→ 用线程组内任一 probe (均匀但不精确)
Level 2: 完全无 probe
→ 上一帧结果 (temporal history)
Level 3: 无历史 (disocclusion)
→ lane intrinsics 取线程组内其他像素均值
Level 4: 全 group 无效
→ 输出黑色 (极罕见)
设计原则: 永远不崩溃, 优雅降级直到最坏输出黑色

I 透明物 / 头发特例 Special

透明物回退
粒子 / 体积雾 / 头发
无表面让屏幕空间 probe 落脚 → 回退到八叉树/网格 irradiance
头发暗化修复
问题: 帽檐暗化但头发未暗 → 头发相对偏亮显灰
解决: 短射线遮蔽 SH
  • 从角色头部内部向各方向发射 0.5m 短射线
  • 忽略背面命中
  • 命中累积成 SH 遮蔽函数
  • 与头发 ambient SH 相乘 → 匹配整体暗化

D 数据流转路径总览 (Data Flow Map)

离线产物 → 运行时
  • Octree 结构 → 运行时查询 (D)、RTGI Miss fallback (Step 4)
  • Probe G-Buffer (BC7) → Octmap Relighting (E)
  • 立体角 LUT → SH 累积 (E, Step 6)
RT 资源 → RTGI Pipeline
  • TLAS + BLAS 池 → Step 3 (TraceRay)
  • Instance Buffer → Step 3 (获取 hit instance) → Step 4 (查 swatch index)
  • Swatch Atlas → Step 4 (hit 着色取漫反射)
RTGI 内部传递
  • Screen Probes → Step 2 (确定方向) → Step 3 (发射) → Step 5 (邻域)
  • Ray Hits → Step 3 输出 → Step 4 输入
  • 中间光照 → Step 4 输出 → Step 5 滤波 → Step 6 时间融合
  • Shadow Rays → Step 3 入队 → 阴影 Pass 执行 → Step 4 使用结果
  • SH 结果 → Step 6 输出 → Step 7 像素插值
跨系统回退路径
  • Octmap SH → Step 7 远距离混合 (>80m)
  • Octmap SH → Step 4 ray-miss 回退
  • Octmap SH → 透明物/头发/粒子 回退
  • 上帧 Irradiance → Step 2 (ray-guide PDF)
  • 上帧 Irradiance → Step 6 (history blend)
  • 上帧 Pixel Result → Fallback Level 2
  • Interior Mask → 选择 interior/exterior SH 组
═══ PERFORMANCE & MODE COMPARISON ═══

P 性能数据 (1440p PS5)

步骤耗时备注
Step 1: Spawn Probes0.15 mscompute shader
Step 2: Ray-Guide0.12 msprefix sum + binary search
Step 3: Cast Rays3.5–5.0 ms~2M rays, 主要瓶颈
Step 4: Light Rays1.0–1.5 ms含 swatch lookup + probe sample
Step 5: Spatial Filter0.8 ms7×7 neighborhood
Step 6: History Filter0.4 ms含 SH 转换
Step 7: Per-Pixel Eval0.5 ms含 distance blend
Octmap Relighting (~1200)0.6–0.8 ms非 RT 模式也运行
RTGI 总计~7–9 ms30Hz RT 模式预算内

Q 渲染模式对比

模式帧率环境光照方案
Performance (非RT)60 HzOctree + Octmap Relighting only
Quality (RT)30 HzOctree + Octmap + RTGI 7-step
设计目标: 两模式视觉必须接近。因为 Octmap Relighting 已经响应动态光源， RT 模式仅增加短距离 (4m) 屏幕空间 GI 细节，二者基底一致。
RT 为何选 GI 而非反射/阴影
选型权衡
  • RT 反射: 古日本场景反射面少, 性价比低
  • RT 阴影: 顶点动画 + cutout 难匹配
  • RTAO: 影响相对小
  • RTGI: 对 BVH/栅格化失配最容忍 → 最终选定
