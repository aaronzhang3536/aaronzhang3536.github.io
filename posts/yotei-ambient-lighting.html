Ghost of Yotei — 环境漫反射光照技术详解

演讲者：Eric Wohllaib（Sucker Punch Productions 图形程序员）| GDC 演讲 | 147页

一、项目背景与挑战

1.1 上一代《Ghost of Tsushima》方案回顾

  • 世界级 probe 网格 + 手工放置的 tetrahedral mesh（四面体网格）用于精细区域
  • 每个 probe 存储天空可见性（sky visibility）的球谐（SH）表示及反弹光照
  • 通过 interior/exterior 标记防止漏光：内部 probe 影响内部表面，外部 probe 影响外部表面

1.2 续作面临的挑战

  1. tetmesh 创作成本高：单建筑 5 小时到 2 天，村落最长 2 周；任何环境改动都需返工
  2. interior/exterior 标记繁琐：组件常跨内外使用，标错率高
  3. 天空可见性 GI 不响应局部光源：内部光照大量靠手工放置 fill light
  4. 首款 PS5 独占需上 RT：cutout 与顶点动画对 RT 性能不友好；目标基础 PS5 60Hz + 30Hz 光追模式，两模式视觉必须接近

二、用八叉树（Octree）替代 Tetmesh

2.1 构建流程

  1. 美术绘制定向包围盒（OBB）
  2. 切分为艺术家指定大小的八叉树根节点（示例 16m）
  3. 包含几何的节点递归细分至最小尺寸（示例 2m）
  4. 几何略微膨胀以保证细分够保守
  5. 每个节点的 8 个角点放置 ambient probe

2.2 嵌入几何的 Probe 处理

  • 烘焙时用双面渲染捕获 probe 周围的 G-Buffer（不翻转背面法线）
  • 若大多数法线指向远离相机方向 → 判定 probe 嵌入几何 → 剔除
  • 剔除后记录相邻 probe 列表，运行时混合邻居数据填充

2.3 八叉树插值（避免不连续）

  • 同尺寸节点：trilinear 混合即可平滑过渡
  • 跨尺寸节点：逐渐 blend 到父节点；强制约束相邻节点尺寸差不超过一级
  • 每节点位字段记录每面/边/角是否通向更大节点
  • 法线偏置：降低位于表面背后 probe 的权重，减少漏光

2.4 边缘混合与多八叉树

  • 八叉树边缘渐隐回退到全局 probe 网格
  • 多八叉树重叠时取均值混合，便于拼出洞穴等不规则空间

2.5 收益

  • 创作时间从天/周降到分钟级
  • 小改不需美术介入，自动每周重烘 2 次
  • 大改只需调整包围盒尺寸
  • 内存开销低，遍历性能合理

三、Interior Masking 改进

3.1 标记来源

  • 整对象标记、shader 参数、deferred decal、顶点画刷
  • 动态对象运行时计算

3.2 Interior Volume Geometry（核心新工具）

美术作一层"建筑墙体内部的水密网格"，类似 stencil shadow 的体积技术：

  1. 绑定深度缓冲 + 全新模板缓冲区
  2. 双面渲染 volume 几何（不绑 RT），仅写模板
  3. 前面 fail depth → stencil +1；背面 fail depth → stencil −1
  4. 最终：stencil = 0 → 外部；stencil ≠ 0 → 内部

支持非凸形状。门窗仍需 vertex paint 实现过渡混合。

3.3 动态对象（角色）

  • CPU 端获取角色所在八叉树节点
  • 每帧发一根物理射线测试 8 个 probe 可见性（一帧一根，缓存结果）
  • 对可见 probe 做 trilinear 混合得到 mask；与上一帧再混合抑制 popping

优点

  • 薄墙友好（适合障子屏风）
  • 灯光可标 interior/exterior 仅照对应表面
  • 八叉树支持任意位置 interior 查询，可驱动风效、玩法

缺点

  • 仅二值，相邻内部空间间无法防漏
  • 标记仍重，美术工作量仍然可观
  • 粒子、体积雾难以精确标记

四、用 Octahedral G-Buffer 替代天空可见性

4.1 动机

  • 天空可见性无法表达局部光源响应
  • 与 RT 路径差异大，难以做到 RT/非 RT 视觉一致

4.2 数据结构

每个 probe 存一个 octahedral 小型 G-Buffer（八叉树 16×16，世界网格 8×8），BC7 压缩：

通道大小
Albedo3 字节
Translucency0.5 字节
Normal2 字节
Surface Distance1 字节
Interior Mask0.5 字节
Sky Visibility1 字节

Translucency 与 Interior Mask 共享一字节互斥编码：内部映射 $[0.5,1]$，外部 translucency 映射 $[0,0.5]$。

4.3 Relighting 流程

  • 每个 texel 视作真实表面：用 distance 重建世界空间位置，用材质数据照亮
  • 方向光阴影：先采样阴影级联；若 texel 落在视锥外，回退到附近 probe 天空可见性方向值；缓存结果避免 popping

4.4 SH 转换：Texel 立体角加权

离线 Python 脚本将每个 texel 近似为两个三角面，按如下公式计算立体角作为 SH 累积权重：

$$\Omega = 2\arctan\!\left(\frac{|\mathbf{a}\cdot(\mathbf{b}\times\mathbf{c})|}{1+\mathbf{a}\cdot\mathbf{b}+\mathbf{a}\cdot\mathbf{c}+\mathbf{b}\cdot\mathbf{c}}\right)$$

4.5 Probe 更新策略

  • 全更新太贵（最坏 12000 个）
  • 采样时打标记；下帧更新超 1 帧未更新且被采样过的 probe，约 1200 个/帧
  • 余量更新更老的 probe；相机切换或新加载强制全量 relight

收益

  • 响应局部光源
  • 美术放灯所见即所得（烘焙几何/材质而非光照）
  • 与 RT hit 着色路径几乎一致 → RT/非 RT 视觉对齐

五、RTGI 增强

5.1 RT 选项权衡

选项问题
RT 反射古日本场景反射面少，性价比低
RT 阴影顶点动画 + cutout 难匹配
RTAO影响相对小
RTGI对 BVH 与栅格化失配最容忍 → 选定方向

5.2 BVH 构建与内容

  • CPU 剔除：复制一份基于 BVH 包围盒的剔除流程
  • GPU 剔除：最终迁移到 GPU，多耗内存但满足性能
  • 仅纳入相机 20m 之外的 LOD；无 RT 专用几何
  • 包含：角色、动画布料、地形、大部分静态几何
  • 不包含：植被顶点动画、非角色 cutout、程序化草/实例、地形洞、水
  • 程序化几何用 imposter 纹理烘到地形 diffuse 近似替代

5.3 内存挑战（约 380MB）

把最大渲染分辨率从 4K 降到 1728p，从渲染目标空隙中挤出空间：

5.4 Material Swatch 系统

灵感来自 Santa Monica Studio 的 Bryan McPhail。每个 swatch 是 32×32 mini-G-Buffer（albedo / translucency / alpha）。

  • 渲染方法：调用常规 vertex shader，但用 UV 当作输出位置（摊平到 UV 空间）；启用保守光栅化确保亚像素三角形输出
  • 像素着色器走正常 deferred 路径，保留层混合、UV 变换等复杂特性；渲染后 dilate 防止 bilinear 越界
  • 每个 mesh+shader 组合共享一份 swatch；含方向相关特性的 shader 按实例独立烘焙
  • 也用于角色 cutout（头发、帽子）
  • 总量上限 5120 个，约 25MB；多数缓存命中，单 swatch 渲染数微秒

5.5 RTGI 七步流水线

Step 1: Spawn Screen-Space Probes

  • 屏幕分 16×16 tile，每 tile 随机选 8×8 象限的一个像素生成 probe
  • 其余象限与已有 probe 比较"覆盖度"（平面距离 + 法线差异），覆盖差则新增 probe
  • 上限 32k 个；1440p 时每 tile 2–3 个

Step 2: Ray-Guided Sampling

  • 上一帧 octahedral irradiance map 作为采样指引（缺历史时用八叉树 octmap）
  • 每 64 线程组：各线程对 texel luminance×N·L 做 InclusivePrefixSum → 随机数二分查找定位方向

Step 3: Cast Rays

  • animated 表面增大最小 hit 距离避免自相交
  • 每 probe 64 根，最大 4m；超出回退世界空间 probe
  • 击中点超出阴影级联 → 入队 shadow ray
  • 最坏约 200 万根/帧

Step 4: Light Rays

  • Hit：从 swatch atlas 取漫反射 + 采样八叉树 → 等效 2 次反弹
  • Miss：回退到八叉树/网格 irradiance（采样位置取 probe 原点，非射线终点，防止漏光）

Step 5: Spatial Filter

  • 每 probe 搜索 7×7 tile 邻域相似 probe（49 线程一线程一 tile）
  • 用 hit distance 重投影命中点位置；atomic 写回任意 texel 保留细节

Step 6: History Filter（Probe 级）

  • 与 AMD GI-1.0 [Boissé '22] 思路一致：$$w = \frac{|L_\text{cur}-L_\text{hist}|}{\max(L_\text{cur},L_\text{hist})}$$
  • 用 lane intrinsics 在 probe 内对所有 texel 取平均 history weight，避免单 texel 噪声破坏整体历史
  • 完成后转换为 SH

Step 7: Final Per-Pixel Evaluation

  • 屏幕分 8×8 tile，预算 min/max 深度，预判八叉树影响 + 是否需 RTGI
  • 80m 内启用 RTGI；远处混入八叉树 SH
  • 像素层做 luminance-delta 历史滤波；剩余噪声交给 FSR3 / PSSR

5.6 Edge-Case 多级回退

  1. 像素无强相关 probe → 用线程组内任一 probe（均匀但不精确）
  2. 完全无 probe → 上一帧结果
  3. 也无历史（disocclusion） → lane intrinsics 取线程组内其他像素均值
  4. 全 group 无效 → 输出黑色

5.7 透明物 / 头发特例

  • 透明物（粒子、体积雾、头发）无表面让屏幕空间 probe 落脚 → 回退到八叉树/网格
  • 头发问题：开 RTGI 后帽檐变暗但头发未变，相对偏亮显灰
  • 解决：从角色头部内部向各方向发射 0.5m 短射线，忽略背面，命中累积成 SH 遮蔽函数；与头发用的 ambient SH 相乘得到匹配整体的暗化

七、总体收益与未来方向

  • 八叉树取代 tetmesh，光照美术工时减少数量级
  • Interior Volume Geometry 将大部分 interior/exterior 标记自动化
  • Octahedral G-Buffer Probe 让 ambient diffuse 响应局部光源、与 RT 视觉一致
  • RTGI 在内存与性能受限下落地，强化过场与光追模式画面质量

未来希望改进：CPU/GPU 剔除路径合并、GPU 剔除几何的 swatch 唯一化、程序化几何的随机体积式 RT 表达、地形洞 cutout、多 interior 之间的漏光防护。
